# WERK-Dokumentation

Die nummerierten Bereiche bilden Vision, Prinzipien, Architektur, Plattform, Module, Entwicklung, Agentenarbeit, Designsystem, API, Sicherheit, Betrieb, Tests und Roadmap ab. Verbindliche Architekturentscheidungen liegen in `adr/`.

Dokumentation und Implementierung werden gemeinsam aktualisiert. Widersprüche werden über eine ADR oder einen klar beschriebenen Task aufgelöst.
