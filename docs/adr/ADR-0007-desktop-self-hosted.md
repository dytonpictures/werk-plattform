# ADR-0007: Desktop-first und Self-hosted-first

- **Status:** Accepted
- **Kontext:** Erste Nutzer bearbeiten komplexe Unternehmensabläufe auf eigener Infrastruktur.
- **Entscheidung:** UX wird primär für Desktop und Betrieb primär für eigene Ubuntu-VMs entwickelt.
- **Begründung:** Passt zum Pilotbetrieb und reduziert frühe Plattformbreite.
- **Alternativen:** Mobile-first, Cloud-SaaS-first.
- **Positive Folgen:** Fokus auf produktive Arbeitsabläufe und Datenhoheit.
- **Negative Folgen:** Native Apps und SaaS-Skalierung folgen später.
- **Sicherheitsauswirkungen:** Betreiber tragen Verantwortung für Host, TLS und Updates.
- **Überprüfung:** Nach erfolgreichem Pilotbetrieb.
