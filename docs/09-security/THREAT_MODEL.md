# Bedrohungsmodell

Zu schützen sind Konten, Berechtigungen, Geschäfts- und Auditdaten sowie Secrets. Relevante Angreifer sind externe Nutzer, kompromittierte Konten und fehlerhafte Erweiterungen. Grenzen liegen an Browser, Reverse Proxy, API, Datenbank und administrativen Skripten.

Das Modell wird vor produktiver Veröffentlichung um konkrete Datenflüsse und Missbrauchsszenarien ergänzt.
